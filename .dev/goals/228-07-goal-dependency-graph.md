# Goal 228 — DevX: `goals/DEPENDENCY.md` 依赖图 + 文件行数硬指标

**Roadmap**: Phase 20 — 0.7 Refactor & Hardening
**依赖**: 无（自包含，loop 可立即执行）
**类型**: C — 元策略/治理（self-improve 主导）

## Why

`ls .dev/goals/ | wc -l` → **229** 个 goal。但 `grep -l "依赖" .dev/goals/*.md` 仅 60+ 个有显式依赖标注——其余 169 个 goal 的执行顺序、合并冲突域、共享文件改动范围**全靠 git log 推断**。

两个具体痛点：
1. loop 调度 4-wide batch 时，无法知道哪些 goal 共享高风险文件（典型例子：tui/app.rs 多 goal 改必然冲突）
2. 仓库行数无硬指标——agent.rs 涨到 2837 行才被发现，而 600 行就该 warn

## Design

### 1. `.dev/goals/DEPENDENCY.md`

```markdown
# Goal Dependency Graph

> Auto-generated by `scripts/build-dep-graph.sh` (last run: <timestamp>).
> Source: each goal's `**依赖**: Goal N, M` line.

## Forward deps (this goal blocks others)

### Goal 219 — Refactor: delete deprecated Agent
- Blocks: 220, 221, 222, 223, 225, 226

## Reverse deps (this goal is blocked by)

### Goal 224 — Lint: deny unwrap_used
- Blocked by: 229-01..229-NN (all unwrap cleanup batches)

## High-risk file map

| File | Touched by goals |
|------|------------------|
| `src/agent.rs` | 219, 222 (final location of moved types) |
| `src/session.rs` | 221 |
| `src/tui/app.rs` | 220, 226 |
| `src/llm/mod.rs` | 222 |
| `src/error.rs` | 223 |
```

### 2. `scripts/build-dep-graph.sh`

```bash
#!/usr/bin/env bash
# 扫描 .dev/goals/ 解析每个文件的 "**依赖**: Goal X, Y" 行
# 输出 .dev/goals/DEPENDENCY.md
# 用 mmdc (mermaid CLI) 渲染 SVG 到 .dev/goals/DEPENDENCY.svg
```

### 3. 文件行数硬指标

`scripts/check-file-sizes.sh`：

```bash
#!/usr/bin/env bash
# 阈值：src/ 下任何 .rs 文件 ≤ 800 行
# tests/ 下任何 .rs 文件 ≤ 1000 行
# 违规：在 CI 输出 WARN 表格，self-improve 时作为 soft-fail
```

阈值参数化：
```bash
MAX_SRC_LINES=${MAX_SRC_LINES:-800}
MAX_TEST_LINES=${MAX_TEST_LINES:-1000}
```

### 4. 与 self-improve.sh 集成

`self-improve.sh` 在 `cargo clippy` 之前调 `check-file-sizes.sh`：
- 任何文件超过阈值 → 软失败（log warning，不 rollback）
- 配合 Goal 220/221/222/225，违反阈值的文件会被拆分 → 阈值是 lower bound

## 验收标准

- `.dev/goals/DEPENDENCY.md` 存在，包含 11 个新 goal（219-229）的依赖图
- `scripts/build-dep-graph.sh` 可重复运行，幂等
- `scripts/check-file-sizes.sh` 在 main 上跑，输出当前仓库的行数表（应有 4-6 个文件超 800 行——这正是 Phase 20 要修的）
- `self-improve.sh` 含 `check-file-sizes.sh` 调用
- `.gitignore` 排除 `.dev/goals/DEPENDENCY.svg`（rebuild 时生成，不入库）

## Non-goals

- 不强制行数违反即 rollback（v0.7 阶段定 soft-fail；v0.8 再考虑 hard-fail）
- 不改 goal 文件格式（继续 `**依赖**: Goal N, M` 文本行）
- 不做 goal 间的 conflict pre-detection（需要 LSP 级集成，scope 之外）
