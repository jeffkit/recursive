## Why manual

`parallel-self-improve.sh deepseek` was attempted twice in this orchestrator
session and failed both times. The wrapper started its `nohup self-improve.sh
… &` child process inside the worktree, but the child shell exited shortly
after constructing the system prompt (around line 112 of self-improve.sh),
before reaching the provider profile / cargo build / agent launch stages.
Foreground `bash -x` reproduction inside the worktree showed the same exit
point, with `+ rm -f $SYSPROMPT_FILE` (the trap EXIT line) firing without
prior error output.

Root cause not isolated. A sibling probe `nohup bash -c 'echo; sleep 30' &`
worked correctly in this same shell environment (PPID reparented to 1, log
flushed, child survived parent exit), so generic nohup detach is not the
culprit. The script's standard launchers work in user's other Codebuddy /
Claude Code sessions on the same repo.

Rather than burn more debug budget on the launcher, this orchestrator
executed the goal scope directly, applying the same quality gates
self-improve.sh would have applied at commit time.

## Lesson recorded

Goal-133 is a second example (after the manual goal-14 / goal-17 instances
recorded in 2026-05-25 journals) of a self-improve commit shipping with
`cargo fmt --all` not having been run. AGENTS.md L154 mandates fmt as part
of every step, but enforcement is by convention only — neither
`self-improve.sh` nor `review-changes.sh` block on `fmt --check`. The
upstream pre-commit gate already includes `cargo clippy -D warnings`, but
fmt is not part of the rollback condition.

Considered but not implemented: extending the rollback condition to include
`cargo fmt --all -- --check` with non-zero exit. Out of scope for this
goal — flagged as a candidate `.dev/`-only improvement.
