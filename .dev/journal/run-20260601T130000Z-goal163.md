# Journal — Goal 163 implementation (2026-06-01)

## Summary

实现 **Goal 163 — TUI 内联到主 crate，成为默认交互入口**，在 main 分支直接提交。

## What was done

### 1. 主 Cargo.toml 新增 tui feature

- 新增 `tui = ["dep:ratatui", "dep:crossterm", "dep:unicode-width"]`
- ratatui / crossterm / unicode-width 标记为 optional 依赖
- `default` features 中加入 `tui`

### 2. 迁移 src/tui/

将 `crates/recursive-tui/src/` 全部迁移到 `src/tui/`：
- 所有 `use recursive_tui::` → `use crate::tui::`
- 所有 `use recursive::` → `use crate::`
- 内部模块路径（`crate::events` 等）→ `crate::tui::events`

新增 `src/tui/mod.rs`（原 lib.rs + TUI 入口 `pub fn run()`）

### 3. src/lib.rs 声明模块

```rust
#[cfg(feature = "tui")]
pub mod tui;
```

### 4. src/main.rs 默认启动 TUI

无参数时：`recursive::tui::run()` 替代原来的 `Cmd::Repl`

### 5. 集成测试迁移

`crates/recursive-tui/tests/backend_smoke.rs` → `tests/tui_backend_smoke.rs`，
加 `#![cfg(feature = "tui")]` gate，import 改为 `recursive::tui::`

### 6. 删除旧 crate

- `crates/recursive-tui/` 目录已删除
- workspace members 移除 `"crates/recursive-tui"`
