- mode:        orchestrator-manual (no LLM)
- baseline:    d55fc7d (post goal-18 merge)
- verdict:     completed
- detail:      Hand-applied the design DeepSeek produced before it got
  Stuck on a 3x-repeated `.into()` patch. The agent's feature design
  (AgentBuilder::seed_transcript, TranscriptFile::take_first_n,
  Replay variant extended with --resume-from + trailing goal) was
  correct. The agent only failed on two mechanical bugs:

  1. `Message::system("sys".into())` is ambiguous because
     `Message::system` accepts `impl Into<String>`; the compiler
     can't infer the intermediate `.into()` target. Manual fix:
     use `.to_string()` explicitly in the new tests.
  2. The transcript.rs tests have three near-identical
     `Message::user("hello".into())` lines and the V4A patch's
     "each context block must be unique" rule fails when the agent
     tries to change all three in one patch without disambiguating
     anchors. The agent retried the same patch 3 times → Stuck.
