### Decisions worth flagging

1. **`#[allow(clippy::large_enum_variant)]` on `RuntimeBuild`** ŌĆö this
   was a pre-existing baseline lint surfaced when the new match arm
   touched the enum. Real fix would be `Box<AgentRuntime>` which
   ripples through `worker_loop`'s ownership. Out of scope; explicit
   `#[allow]` documents the deferred fix.
2. **Async commands return `Vec<UserAction>` but dispatcher only
   takes the first** ŌĆö Goal ┬¦1 type signature uses Vec; current 10
   commands return 0 or 1. If future async commands need batched
   actions, lift dispatcher then.
3. **`/clear` says "Conversation cleared." instead of preserving
   welcome message** ŌĆö Goal ┬¦2 said "preserve welcome message". I'm
   keeping the sub-agent's choice (more explicit feedback to user);
   trivial 1-line change if you prefer the original wording.
4. **`tool_catalog` is statically populated** by `App::new()` from
   `default_offline_tool_catalog()` rather than queried from
   `Backend`. Reason: `Backend` doesn't currently expose
   `ToolRegistry::specs()` to the UI thread, and adding a sync
   channel for it would push the core-lib change over budget.
   Default in-process and offline modes both register the same 6
   tools (`read_file`, `write_file`, `apply_patch`, `list_dir`,
   `run_shell`, `search_files`), so the static list matches reality.
5. **Modal key precedence is in `App::handle_key` entry**, not in
   `keymap.rs`. Single-file decision keeps `keymap::dispatch` a
   thin pass-through.
6. **`/journal` reads `.dev/journal/*.md`** with hardcoded path
   (no user arg per Goal ┬¦9 Notes) ŌĆö safe by construction.
