## Design decisions

- `GoalEvaluator` sends only the last 20 transcript messages to keep evaluation cheap.
- `run_goal_loop` is an `AgentRuntime` method (not a new branch in the agent loop),
  preserving Invariant #1.
- The HTTP `POST /sessions/:id/goal` handler locks the per-session Mutex briefly to call
  `set_goal` (write only to the shared `Arc<RwLock>`), then returns immediately — the
  loop is not driven by HTTP but the caller can monitor via SSE.
- `GoalCleared` is emitted both on explicit clear, turn-budget exhaustion, and on
  condition achieved (the loop clears itself after success).
