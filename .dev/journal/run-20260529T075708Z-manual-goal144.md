## Result

Goal 143 left the TUI with single-line `You: ...` / `Agent: ...` rendering.
Goal 144 replaces that with bordered, blockified messages, a multi-segment
status bar, streaming partial-token assistant output, an `apply_patch`
diff view, and `Ctrl+E` expand for tool results.

### What landed

- `crates/recursive-tui/src/app.rs`: replaced `Vec<StyledMessage>` with
  `Vec<TranscriptBlock>` (8 variants: User / Assistant / ToolCall /
  ToolResult / Diff / Compacted / System / Error). Added `UsageStats`
  (token + cost accumulators), `TurnState` (running flag + start instant
  + verb), and turn counter. ~1164 lines (was ~624).
- `crates/recursive-tui/src/backend.rs`: `TuiEventSink` now maps 7 of
  11 `AgentEvent` variants (`PartialToken`, `AssistantText`, `ToolCall`,
  `ToolResult`, `Usage`, `Latency`, `Compacted`). The remaining 4
  (`PlanProposed`/`PlanConfirmed`/`PlanRejected`/`TurnFinished`-passthrough)

  `UiEvent::TurnFinished` after `runtime.run()` returns to guarantee
  spinner shutdown.
- `crates/recursive-tui/src/events.rs`: 9 `UiEvent` variants now —
  `AssistantPartial`, `AssistantMessage`, `ToolCall {id, name, arguments}`,
  `ToolResult {id, name, output, success}`, `Usage`, `Latency`,
  `Compacted`, `TurnFinished`, `Error`.
- `crates/recursive-tui/src/keymap.rs`: `dispatch` now takes the full
  `KeyEvent` (so we can read `KeyModifiers::CONTROL`). Added handling
  for `Ctrl+E` (toggles last `ToolResult`/`Diff` `expanded` flag).
- `crates/recursive-tui/src/main.rs`: passes the whole `KeyEvent`,
  feeds spinner frame index per draw.
- `crates/recursive-tui/src/ui/transcript.rs` **(new, 409 lines)**:
  per-block render functions; truncates tool output to 6 lines with

- `crates/recursive-tui/src/ui/diff.rs` **(new, 103 lines)**: parses
  V4A `apply_patch` input → `DiffHunk { lines: Vec<DiffLine> }`,
  renders +/- in green/red, file header in bold.
- `crates/recursive-tui/src/ui/spinner.rs` **(new, 44 lines)**: 10-frame
  braille animation, advances every draw, emits

- `crates/recursive-tui/src/ui/status.rs` **(new, 173 lines)**: 5-segment

### Decisions worth flagging

1. **`UiEvent::TurnFinished` is synthetic, not just a 1:1 mapping of
   `AgentEvent`.** The worker emits it unconditionally after
   `runtime.run()` returns, so spinner shutdown is provider-agnostic.
2. **`apply_patch` Diff block is generated at `ToolCall` time**, not
   `ToolResult`. The V4A patch is fully visible in the call args, so
   showing the diff immediately (rather than waiting for the result)
   matches user expectation. The `✓ apply_patch` confirmation block
   still appears after.
3. **`write_file` Diff is a stub** — just shows `📝 Updated path/to/file
   (N bytes)` without diffing the actual content (would require reading
   the previous version). Goal §4 explicitly allowed the fallback.
4. **Streaming integration test tolerates missing `AssistantPartial`.**
   The runtime's forwarder task (`runtime.rs:191`) and the main task
   race on the unbounded mpsc; depending on scheduling, MockProvider's
   single `tx.send(content)` may complete before any partial is
   forwarded. The test asserts `AssistantMessage` + `TurnFinished` are
   present, partials are best-effort.
5. **Turn count is locally accumulated** in `App`, not pulled from
   `runtime.turn_index()` — runtime lives in the worker task, exposing
   it would require a new sync channel. Out of goal 144 scope.
6. **Pricing table is hardcoded for 4 models**, no env override. If the
   model isn't in the table, the cost segment is omitted. Adequate for
   the common case; richer pricing config can land later.
