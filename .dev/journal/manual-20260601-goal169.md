**Date**: 2026-06-01

YAML frontmatter from `.recursive/skills/` and `~/.recursive/skills/`, making them
available as `/command-name` aliases.

## Design decisions

- `CommandSpec` uses `&'static str` for all fields (born-static built-ins). Rather than
  change this, `CommandRegistry` now holds a separate `Vec<SkillCommand>` for dynamic
  skills. Built-ins are always checked first; skills are a fallback.
- `SkillCommandLoader::load()` is synchronous (filesystem scan at startup) to keep the
  async worker loop free of I/O blocking.
- Skill files without a `name:` frontmatter key derive the name from the filename stem.
- Argument substitution uses `$ARGUMENTS` placeholder (case-sensitive).
