## Result

ROADMAP-v4 17.5 was 🔴. g138 ships:

### `Dockerfile` (repo root, ~75 lines)

Multi-stage:

- `builder` from `rust:1.86-slim`. Uses BuildKit cache mounts on
  `/build/target`, `/usr/local/cargo/registry`, `/usr/local/cargo/git`
  for fast iteration without bloating the final image.
- `runtime` from `debian:bookworm-slim`. Adds only `ca-certificates`
  via apt (needed for outbound HTTPS to LLM providers); `wget` is
  preinstalled and used for HEALTHCHECK.
- Non-root `recursive` user (uid/gid 1000) — k8s securityContext
  default.
- `EXPOSE 3000`, `HEALTHCHECK` against `/health`, `ENTRYPOINT
  /usr/local/bin/recursive`, `CMD ["http", "--addr", "0.0.0.0:3000"]`.

Build target `--bin recursive` (not the whole workspace) skips
examples + extra binaries → faster + smaller artifacts.

### `.dockerignore` (repo root)

Mirrors `.gitignore` plus excludes: `.dev/`, `.github/`, `.worktrees/`,
`e2e/`, `tests/`, `docs/`, `examples/`, all `.md`, editor state
(`.codebuddy/`, `.cursor/`, `.idea/`, `.vscode/`), env files. Keeps
the build context small.

### `.github/workflows/docker-image.yml`

Triggers: tag push `v*.*.*` and `workflow_dispatch`. Uses
`docker/metadata-action` for tag derivation, builds multi-arch
(`linux/amd64,linux/arm64` via QEMU), pushes to
`ghcr.io/<owner>/recursive` with semver, `<major>.<minor>`, and
`latest` tags. GHA cache (`type=gha`) for cross-run dep cache reuse.

## Issues encountered & resolved

1. **First Dockerfile attempt used a "stub main + restore" cache trick
   that didn't account for `[lib]`.** The crate is both a lib and
   a bin (`Cargo.toml` declares both); `echo 'fn main() {}' > src/main.rs`
   is enough for a pure-bin crate, but cargo also wants `src/lib.rs`
   to exist. The stub trick would have needed a fake lib.rs too,
   plus stubs for every `[[example]]` declared in `Cargo.toml`
   (basic, with_tools, with_hooks, with_mcp, etc.). Net complexity
   not worth it — switched to BuildKit cache mounts on
   `target/` and the cargo registry. That gives the same speed
   benefit with one fewer source-of-truth.

2. **Default CMD `http` requires API key env to actually serve.**
   `recursive http` calls `Config::require_api_key()` at startup
   (src/main.rs:341); without `RECURSIVE_API_KEY` (or one of the
   provider-specific env vars), the binary exits with a clear
   error. This is correct product behavior — flagged in the journal
   so operators know the deploy needs at minimum one env var. The
   image's HEALTHCHECK doesn't break on this because it runs once
   the process is up (and a process that exited has no
   HEALTHCHECK firing anyway).

## Design decisions captured here

- **`bookworm-slim` not distroless.** Distroless static would have
  no shell for `HEALTHCHECK CMD wget …`; distroless glibc would
  add a layer with marginal size benefit. `bookworm-slim` is
  ~75 MB and trivially update-able.
- **Single `/health` endpoint, no `/livez` vs `/readyz` split.**
  The agent server is stateless from a probe perspective —
  "process up" implies "ready to serve". A future goal could
  introduce a `/readyz` that gates on "MCP servers spawned and
  reachable" if that becomes a real failure mode.
- **GHCR over Docker Hub.** No extra secret needed (`GITHUB_TOKEN`
  is auto-provisioned), bound to the GitHub repo, free for public
  packages, supports OCI properly. Switching is one-line if the
  repo wants to mirror.
- **Multi-arch via QEMU.** First arm64 tag-build will be slow
  (~10-15 min in CI); cache-from gha keeps subsequent builds
  fast. If this proves too slow, the alternative is a matrix
  per-arch + manifest merge — out of scope for this goal.

## Follow-ups recorded

- **First-time UX**: a deploy that doesn't set `RECURSIVE_API_KEY`
  gets a fast-fail with a clear message (good), but a `docker
  compose` example or quickstart in README would help. Out of
  scope for g138; flag as a docs improvement.
- **Image security scan**: a future `.github/workflows/docker-scan.yml`
  could run trivy on the image to flag CVEs in
  `bookworm-slim`/`ca-certificates`. Out of scope.
- **`recursive run` from a one-shot container**: today the CMD is
  `http`. If a user wants `docker run recursive:test run "some
  goal"`, they need to pass the run subcommand. Document this in
  README later.
