## Result

User reported four issues after running the post-Goal-149 TUI:

1. Status bar shows `gpt-4o-mini` even though `~/.recursive/config.toml`
   says `deepseek-v4-flash`
2. Transcript can't scroll up — long conversations are stuck at bottom
3. Palette feels too dim on dark terminals (DarkGray gutters, Cyan
   body)
4. Markdown is not rendered — `## Heading`, `**bold**`, `` `code` ``
   show their literal markers

Goal 150 fixes all four.

### What landed

**Fix 1 — model name from config**

`crates/recursive-tui/src/app.rs::detect_model_name`:

- Same priority chain `Backend::build_runtime` uses (Goal 149):
  `RECURSIVE_MODEL` → `OPENAI_MODEL` → `~/.recursive/config.toml`
  `[provider].model` → `gpt-4o-mini`
- Uses `recursive::config_file::FileConfig::load()` (already public)
- Empty model strings in config fall through to the next source

**Fix 2 — real scroll cap**

`crates/recursive-tui/src/ui/chat.rs`:

- ratatui 0.29's `Paragraph::line_count` is unstable, so we approximate:
  `total_rows = sum over lines of (line.width().div_ceil(inner_width)).max(1)`
- `Line::width()` is unicode-aware and public
- This is the count *after* word-wrap, so PageUp/↑ now actually
  scrolls long messages all the way to the top
- Empty lines still count as 1 row so blank separators don't collapse

**Fix 3 — palette brightening**

- `Color::DarkGray` → `Color::Gray` everywhere in transcript / diff /
  input footer
- User block label: `Color::White` → `Color::LightBlue`
- Assistant block label: `Color::Cyan` → `Color::LightCyan`
- Assistant body text: was solid `Color::Cyan`, now `Color::White`
  default with markdown-driven highlights
- Note-mode `#` indicator kept as `DarkGray` (intentionally subtle —
  Note is the "muted" mode)

**Fix 4 — basic markdown renderer**

New file `crates/recursive-tui/src/ui/markdown.rs` (~280 lines):

- `MdState { in_code_block }` carried across consecutive lines so
  fenced code blocks span multiple lines
- `render_inline(line, default_fg, state) -> RenderedLine { spans, state }`
- Constructs:
  - `**bold**` → LightCyan + BOLD
  - `*italic*` / `_italic_` → default_fg + ITALIC
  - `` `code` `` → LightYellow
  - `# heading` ... `###### heading` → LightCyan + BOLD, hashes stripped
  - `- ` / `* ` / `+ ` bullet → LightYellow `• ` + body, indent preserved
  - ` ``` ` fenced code block → entire block in LightYellow, parser
    skips inline rules
- One-pass byte-level state machine, no regex
- Tolerant: unmatched markers render as plain text (LLMs emit
  malformed markdown all the time)
- 9 unit tests

`render_assistant` in `transcript.rs` now feeds each text line through
`markdown::render_inline`, threading `MdState` so a fenced block
opened on one line still gets code styling on subsequent lines.

### Decisions worth flagging

1. **`Paragraph::line_count` is unstable in ratatui 0.29**, so the
   wrapped-row count is approximated via `Line::width()`. This is
   slightly conservative for inputs with combining characters or
   double-width emoji that wrap mid-cluster, but:
   - the worst case is a one-row over-estimate, which means we let
     the user scroll one row past the top, harmless
   - matches what ratatui's own `line_count` stub does internally
2. **Markdown is one pass, byte-level**, not a CommonMark parser.
   Tables, links, blockquotes, escapes, reference-style links all
   out of scope. If a future LLM emits e.g. a markdown table, it
   shows up as monospace plain text — not pretty but readable.
3. **Note `#` indicator stays DarkGray** because Note is the "muted
   mode" (not LLM, not shell, just a local annotation); a brighter
   colour would compete with Bash's LightYellow.
4. **Code block fence rendering**: the fence line itself shows as
   `Color::Gray`, body as `LightYellow`. fake-cc renders fences
   completely invisible — we keep them visible because in a TUI
   the visual demarcation helps comprehension.
5. **Empty model in config.toml** (e.g. `model = ""`) is treated as
   "absent" and falls through to the default, not silently shown.
6. **HOME-dependent test consolidation**: the new
   `detect_model_name_falls_back_to_config_file` test grabs the same
   `PinnedHome` mutex as `backend::offline_mode_and_config_file_resolution`
   from Goal 149, so they serialise automatically — no flakes.
