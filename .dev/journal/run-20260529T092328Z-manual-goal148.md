### Decisions worth flagging

1. **626 lines instead of goal §6's "1500-3000 line" guidance.** The
   sub-agent's call: 314 comparison rows is exhaustive coverage of
   what's actually visible in fake-cc; padding to 1500+ would mean
   repeating rationale for each ⛔ row or splitting fine-grained
   sub-rows that aren't load-bearing. The doc's value is the row
   count and citation density, not its line count. I accepted this
   trade-off.
2. **Added a 12th "维护备注" section** (not in goal §2's 8+2 list).
   46 lines explaining when to re-evaluate, citation hygiene, and
   how to downgrade a status if a feature regresses. Useful enough
   to keep; trivial to delete if reviewers prefer.
3. **Markdown / syntax highlighting marked 🔴 (not 🟡)**. Recursive's
   Assistant block has Cyan colouring but no real markdown rendering
   (no bold/italic/list/link). Calling that 🟡 would inflate the
   alignment percentage; honest 🔴 is more useful as a future TODO
   marker.
4. **Esc double-press marked 🟡 (not ✅)**. fake-cc escalates the
   second press to quit; Recursive's Goal 147 chose "Esc never
   quits" (table cell explicitly notes the divergence rather than
   pretending it's aligned).
5. **`/journal` is described as relying on `.dev/journal/`** which
   is a development-only directory. Production users won't have it.
   Doc notes this in §6 备注.
