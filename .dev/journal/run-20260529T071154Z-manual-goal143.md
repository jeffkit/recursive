# Manual landing of goal-143 (TUI revamp step 1: skeleton + in-process runtime)

- goal source:  .dev/goals/143-tui-revamp-skeleton.md
- mode:         orchestrator-manual (sub-agent execution, manual review)
- branch:       feat/tui-revamp-step1
- verdict:      completed

## Result

`crates/recursive-tui/src/main.rs` was a 958-line single-file TUI that
talked to the agent over `reqwest` HTTP polling. This goal restructured
it into modules and switched the transport to in-process via
`AgentRuntime`, with no behavioural change visible to the user.

### What landed

- `Cargo.toml`: dropped `reqwest`; added `[lib]` block to expose modules
  to integration tests; added `async-trait` dep (already in root
  Cargo.toml — not a new project-level dep) for the `EventSink` trait;
  dev-dep `recursive-agent` now enables `test-utils` feature for
  `MockProvider`.
- `src/main.rs`: 958 → **66 lines**. Pure entry: terminal init,
  `Backend::spawn()`, tokio select! loop, terminal restore.
- `src/lib.rs`: 11 lines, re-exports of the five modules.
- `src/app.rs`: 624 lines. `App`, `AppScreen`, `StyledMessage`,
  `handle_key` (synchronous now — returns `Option<UserAction>`),
  `handle_ui_event`, scroll. 34 unit tests migrated from `main.rs`.
- `src/events.rs`: 38 lines. `UiEvent` (was in `main.rs`), new
  `UserAction` enum (`SendMessage`, `ConfirmPlan`, `RejectPlan`,
  `Shutdown`).
- `src/keymap.rs`: 45 lines. `dispatch(&mut App, KeyCode) ->
  Option<UserAction>` thin shell. 2 tests.
- `src/backend.rs`: 284 lines. `Backend` (action_tx + event_rx +
  worker JoinHandle), `TuiEventSink: EventSink` mapping 4 of 11
  `AgentEvent` variants (the same 4 the old TUI consumed),
  `RuntimeBuild::{Ready,Offline}` for graceful degradation when no
  API key is set, `worker_loop` with single-task ownership of the
  `&mut AgentRuntime`. 1 test.
- `src/ui/{mod,splash,chat,plan_review}.rs`: 23 + 41 + 58 + 41 lines.
  Pure render functions, semantic-identical output.
- `tests/backend_smoke.rs`: 72 lines. `MockProvider` round-trip — sends
  `UserAction::SendMessage("hello")`, asserts a `UiEvent::AssistantMessage`
  arrives within 5s.

Total: 1303 lines across 11 files (was 958 in 1 file). Code grew because
the worker task + EventSink + offline degradation + module boundaries are
not free, but each module is small and single-purpose.

### Decisions worth flagging for future steps

1. **`#[lib]` block added** — goal §2 didn't list a `lib.rs`, but Rust
   binary crates can't be imported by `tests/*.rs` without a library
   target. This is the minimum-mechanism way to satisfy goal §5
   (integration test in `tests/backend_smoke.rs`).
2. **EventSink mapping is conservative** — only 4/11 `AgentEvent`
   variants are forwarded right now, matching the old TUI's
   functional surface. Goal 144 (transcript blocks) will widen this
   to consume `PartialToken` / `Usage` / `Latency` / `Compacted`,
   and Goal 147 (plan protocol) will pick up `PlanProposed` /
   `PlanConfirmed` / `PlanRejected`.
3. **`UserAction::ConfirmPlan` calls `runtime.run("")` after
   `confirm_plan()`** — this is goal-prescribed and matches the
   pre-revamp TUI's behaviour. The empty-string turn lets the kernel
   resume from `PlanPending` state. If empty turns become noisy in
   logs we'll revisit.
4. **`RECURSIVE_TUI_MOCK=1` env hook mentioned in goal §3 was NOT
   wired** — the goal called it "(mostly useful for tests; the
   integration test wires a richer mock via the
   `recursive-agent/test-utils` feature)". Since the integration
   test does exactly that, the env hook would have been dead code.
   Skipped intentionally.
5. **`handle_key` is now sync, returning `Option<UserAction>`** —
   the old `async fn handle_key(KeyCode, &Sender<UiEvent>)` mixed
   state mutation with side effects. Decoupling is cleaner: state
   updates happen inline, side effects (network/runtime work) get
   dispatched to the worker via `action_tx`. All 34 old keymap
   tests still pass.
