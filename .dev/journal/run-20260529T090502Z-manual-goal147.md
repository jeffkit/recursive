## Result

Replaces the legacy `"plan:"` / `"## plan"` text-prefix hack (introduced
as a TUI hack in goal 143) with the structured
`AgentEvent::PlanProposed/Confirmed/Rejected` protocol that the kernel
already emits. Adds double-press semantics for `Esc` and `Ctrl+C` so
neither key accidentally exits during a long turn.

### Double-press semantics summary

**`Ctrl+C`**:

| state | first press | second press (within 2s) |
|---|---|---|
| Modal open | pop top modal | quit |
| No modal, buffer non-empty / non-Prompt | clear buffer | quit |
| No modal, buffer empty, turn running | push "Interrupting…" + dispatch `UserAction::Interrupt` | quit |
| No modal, buffer empty, idle | push "Press Ctrl+C again to exit" | quit |

**`Esc`** (NEVER quits):

| state | action |
|---|---|
| PlanReview modal | dispatch `UserAction::RejectPlan("user rejected")` + close modal |
| Other modal | pop top modal |
| No modal, buffer non-empty / non-Prompt | clear buffer |
| No modal, buffer empty, turn running | dispatch `UserAction::Interrupt` |
| No modal, buffer empty, idle | no-op |

### Decisions worth flagging

1. **`Modal::PlanReview.tool_calls` is `Vec<serde_json::Value>`**, not
   `Vec<recursive_agent::llm::ToolCall>` as goal §1 sketched. Reason:
   `AgentEvent::PlanProposed.tool_calls` is actually
   `Vec<serde_json::Value>` in `src/event.rs:80-83` (each value is
   `{name, id, arguments}`). The goal forbade modifying
   `src/event.rs`, so the TUI accepts whatever the kernel sends.
   `render_plan_review` reads `tc.get("name")` / `tc.get("arguments")`
   defensively.
2. **Removed 6 pre-existing tests** that asserted the deleted behaviour
   (`plan_message_triggers_plan_review`, `plan_edit_prefills_input`,
   `esc_sets_should_quit_when_buffer_empty`, etc.). Replaced by the 13
   goal-§7 tests covering the new protocol.
3. **Cancel test asserts `wait_for_cancel(true)` returns quickly**
   rather than racing a long Mock turn. Constructing a 5s-blocking mock
   provider and racing `Interrupt` against it in cargo test's
   parallelism is flakey (cf. `.dev/AGENTS.md` reqwest deadlock note).
   The pair `interrupt_action_sets_cancel_flag` +
   `run_with_cancel_flag_true_returns_quickly` covers the same chain
   (flag flip → select arm wakes → turn aborts) deterministically.
4. **`edited_text` in `Modal::PlanReview` is currently always `None`**.
   The `e` key copies `plan_text` to `app.prompt.buffer` and closes
   the modal — there's no inline-edit UX in this step. The field is
   kept because goal §1 listed it explicitly and it's free to read
   later.
5. **Esc never escalates on second press**. Goal §5 says "2 秒内再次按
   仍然不退出. Esc 不再是退出键." Implemented literally; double-press
   tracker still records the timestamp for any future feature, but
   nothing reads it for Esc.
