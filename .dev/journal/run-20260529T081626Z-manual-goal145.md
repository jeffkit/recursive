## Result

Replaces the simple `App.input: String` with a 4-mode PromptInput state
machine matching fake-cc's `inputModes.ts` design: Prompt (default) /
Bash (`!` prefix → run_shell tool, no LLM) / Note (`#` prefix, local
only) / Command (`/` prefix, dispatch deferred to goal 146).

### `InputMode` state-machine truth table

| from | event | to |
|---|---|---|
| Prompt (buf empty) | type `!` | Bash (buf empty, prefix consumed) |
| Prompt (buf empty) | type `#` | Note (buf empty) |
| Prompt (buf empty) | type `/` | Command (buf empty) |
| Prompt (buf empty) | type any other char `c` | Prompt (buf = `c`) |
| Prompt (buf NOT empty) | type `!`/`#`/`/` | Prompt (chars enter buf normally) |
| any non-Command | `Shift+Tab` | cycle Prompt → Bash → Note → Prompt (skips Command) |
| Bash/Note/Command (buf empty) | `Backspace` | Prompt |
| any (buf NOT empty) | `Backspace` | delete previous char, mode unchanged |
| any | `Esc` (buf empty + Prompt) | quit |
| any | `Esc` (else) | clear buf + reset to Prompt (does NOT quit) |
| any | `Enter` submit | clear buf, cursor=0, **force mode=Prompt**, push history |
| Prompt (history_idx=None) | `↑` (buf empty + history non-empty) | enter history nav, load latest entry, restore its mode |
| in history nav | `↑` | move further back, restore mode of that entry |
| in history nav | `↓` | move forward; past newest → restore draft + draft_mode |

### Decisions worth flagging

1. **`run_shell` arg is `command`, not `cmd`** — Goal §7 sample code
   used `cmd` but `RunShell::execute` reads `args["command"]`
   (`src/tools/shell.rs:74`). Followed the real schema; sub-agent
   caught this.
2. **Bash mode uses an independent `ToolRegistry`**, not
   `runtime.kernel().tools()`. Reasons: (a) `RuntimeBuild::Offline`
   has no runtime, but bash mode must still work; (b) decouples
   bash side-effects from agent transcript state.
3. **History `↑` allowed even when buffer is non-empty *if* already
   navigating history.** Goal §5 said "buffer empty only", but
   that breaks consecutive `↑` (after first press buffer is non-empty
   so second `↑` would go to scroll). Pragmatic relaxation; covered
   by `history_up_down_navigates_records`.
4. **`Esc` no longer quits unless buffer is empty AND mode is Prompt
   AND no modal is open.** This is the "fake-cc" behaviour; quit is
   moved to `Ctrl+D` / `q` (in modals) / `/exit`. The Goal 147
   double-press logic will refine this further.
5. **`Ctrl+E` dispatch is in `App::handle_key`**, not in `keymap.rs`,
   because the fork (input nav vs transcript expand) depends on app
   state. `keymap.rs` remains a thin pass-through.
6. **No `unicode-segmentation` dep** — used stdlib `char_indices()` /
   `len_utf8()`; multi-byte cursor handling tested with
   `cursor_handles_multibyte_chars` (Chinese input).
